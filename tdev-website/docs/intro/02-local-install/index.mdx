# Lokale Installation

<Steps>
1. Node Installieren
    - Installation WSL2 + Node.js + (optional) Docker: [WSL2-Installation](./WSL2/index.mdx)
    - Node unter Windows installieren: [Node-Installation](./Windows/index.mdx)
    - Node unter macOS installieren: [Node-Installation](./macOS/index.mdx)
    :::tip[Entscheidungshilfe unter Windows]{.compactDeflist}
    Lokal
    : **Vorteil**: schnell und einfach, besonders für die *Offline-Nutzung* ohne API-Anbindung geeignet.
    : **Nachteil**: API-Anbindung ist schmerzhaft - besser auf WSL ausweichen.
    WSL2
    : Windows Subsystem for Linux (WSL)
    : **Vorteil**: Bessere Linux-Kompatibilität, besonders für die *API-Anbindung* geeignet.
    : **Nachteil**: Etwas komplexer bis die Voraussetzungen installiert sind, danach für jedes weitere Projekt einfach einzurichten.
    :::
2. **Repository klonen** - das Repository von Github klonen und in VS Code öffnen.
3. Abhängigkeiten installieren: In VS Code __Terminal > Neues Terminal__ öffnen und folgenden Befehl ausführen:
    ```bash
    yarn install
    ```
4. `.env` hinzufügen
    ```env title=".env"
    OFFLINE_API="indexedDB"
    ```
    :::info[Hinweis]
    Die `.env`-Datei ist notwendig, um die API-Anbindung zu deaktivieren.
    Sie sollte nie in einem öffentlichen Repository hochgeladen werden, da sie sensible Informationen enthalten kann.
    :::
5. Entwicklungsserver starten: Im Terminal folgenden Befehl ausführen:
    ```bash
    yarn run start
    ```
</Steps>
